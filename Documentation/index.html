Neo-Blog Project Documentation

A comprehensive guide to understanding, modifying, and deploying the Nirmal Magar personal blogger site.

1. Introduction

Welcome to the Neo-Blog documentation. This project is a premium, beautifully crafted personal portfolio and blog site designed for Nirmal Magar (@iam-neo).

It utilizes the modern React ecosystem, prioritizing performance, SEO, and visual aesthetics through glassmorphism, dynamic animations, and dark-mode optimization.

2. Key Features

  • Blazing Fast Performance: Fully statically generated (SSG) utilizing Next.js App Router.
  • MDX Blog System: Write posts in Markdown while seamlessly embedding React components.
  • Premium UI/UX: Dark-themed aesthetic featuring glassmorphism, Tailwind CSS styling, and Framer Motion micro-interactions.
  • SEO Optimized: Built-in dynamic Open Graph tags, canonical URLs, and structured metadata.
  • Responsive Layout: Flawless rendering across Mobile, Tablet, and Desktop resolutions.

5. Writing Blog Posts

The blog is powered by local `.mdx` files. To create a new post, simply follow these steps:

  1. Navigate to the src/content/posts/ directory.
  2. Create a new file with a .mdx extension (e.g., my-new-post.mdx).
  3. Add the necessary YAML frontmatter at the very top of the file:
---
title: "Your Awesome Blog Post Title"
date: "2026-02-22"
excerpt: "A brief, compelling summary of what the post is about."
tags: ["react", "web-dev", "tips"]
---

# Hello World

Here is where your Markdown content goes. You can write **bold text**, *italic text*, and use code blocks.

Once you commit and push this new file to GitHub, the site will automatically rebuild on Netlify and the new post will appear in the /blog feed.

6. Key Components

  • lib/posts.ts: Parses MDX files from the filesystem, extracts the frontmatter using gray-matter, calculates reading time, and handles related posts logic.
  • app/blog/[slug]/page.tsx: Dynamically generates static pages (SSG) for every post utilizing generateStaticParams().
  • components/mdx-components.tsx: Maps standard HTML tags inside Markdown to custom, styled React components ensuring the blog text matches the site's premium aesthetic.

7. Deployment

The application is pre-configured for seamless deployment to Netlify.

  1. Push the repository to GitHub.
  2. Log into your Netlify dashboard and click "Add new site" -> "Import an existing project".
  3. Select the GitHub repository.
  4. Netlify will automatically detect the netlify.toml configuration file.
  5. Click Deploy site.

The build command executed will be npm run build, and the output directory deployed will be .next.

8. How to Edit Code

This section provides guided instructions on how to modify dynamic content sections such as Blog Series and the interactive About page journeys.

Managing Blog Series

You can group related blog posts into a numbered series. This automatically links them together at the bottom of each post.

  1. Adding to an Existing Series: Open the .mdx file of your new post. Add two lines to the frontmatter: 
    series: "Name of Existing Series"
seriesPart: 3
    Make sure the seriesPart number corresponds to the logical order in the series.
  2. Creating a New Series: There is no centralized registry to create a series. You simply define a new series name in the frontmatter of any post. 
    series: "My Brand New Series"
seriesPart: 1
    The system will automatically detect the new series, generate an index page for it, and list it globally.

Editing "Choose Your Adventure" (About Page)

The interactive journey selector on the About page is driven by a hardcoded configuration object.

  1. Navigate to src/components/interactive-path-selector.tsx.
  2. Locate the const paths object near the top of the file.
  3. To add a new article to a path (e.g., frontend), simply add an object to the articles array: 
    articles: [
    { title: "New Blog Post", url: "/blog/new-post-slug", readTime: "5 min" },
    ...
]
  4. To add an entirely new journey path, add a new key to the paths object with the required id, label, icon, color, bg, description, and articles properties.
